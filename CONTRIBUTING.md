# Contributing Guidelines

First off, thank you for considering contributing to Gitmemoji. An active
community is what makes it a great tool for learning Gitmoji. There are various
ways in which you can contribute to the project:

## Reporting bugs

If you find a security vulnerability, do NOT open an issue. Please read the
[security policy](SECURITY.md) instead.

You can report bugs by creating an issue with the bug report template [here](https://github.com/Lezurex/gitmemoji/issues/new/choose).
Please make sure to include all the details for developers to be able to reproduce
the bug.

## Suggesting new features

Have an idea for improving Gitmemoji? We'll be happy to hear it! Submit a new
issue with the feature request template [here](https://github.com/Lezurex/gitmemoji/issues/new/choose)
and tell us what your feature is, why you need it and how it should work.

## Contributing code

If you have some experience with JavaScript, TypeScript and React, you can help
us by contributing code. Take a look at the open issues for inspiration, fork
the repo, make your changes in a separate branch and create a new pull request.
It'll then be reviewed and eventually merged in a few days.
